# STM32 Central Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create `stm32_central/` — a second B-L475E-IOT01A project that acts as BLE GAP Central, scanning for "BlueNRG", subscribing to AccGyroMag notifications, printing IMU data over UART, and accepting `freq <n>` commands to write sampling frequency.

**Architecture:** Full copy (Approach B-2) of the server project into `stm32_central/`. Server-only files (gatt_db, sensor) are removed. New files (central.h, central.c) implement the BLE state machine and UART command parser. app_bluenrg_ms.c is replaced with a central-role init.

**Tech Stack:** BlueNRG-MS SDK on B-L475E-IOT01A, STM32L4 HAL, CMake
